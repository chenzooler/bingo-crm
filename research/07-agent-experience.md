# חוויית הנציג — מסמך UX

> איך בונים מערכת שנציג חדש לומד בשעה ועובד עליה כל היום בלי תסכול.

---

## עקרונות מנחים

### 1. "Tab Once, Save Forever"
כל שדה נשמר אוטומטית כשהנציג עובר ל-tab הבא. אין כפתור "Save". אין modals של "האם לשמור?".

### 2. "AI shows, Agent decides"
AI מציע - לעולם לא פועל אוטונומית בלי אישור. המידע תמיד שקוף לנציג.

### 3. "The customer is the hero"
המסך עוצב כאילו הלקוח עומד מאחורי הנציג ורואה - לכן ניסוחים נעימים, אייקונים יפים, לא בירוקרטיים.

### 4. "Don't make me think"
ברירות מחדל חכמות. אם הנציג צריך לקבל החלטה, היא חייבת להיות ברורה. אם לא צריך - לא שואלים.

### 5. "Keyboard is king"
כל פעולה חוזרת = shortcut. נציג טוב לא מזיז עכבר.

**08:30** - נציג מגיע, פותח את ה-CRM

**09:00** - מתחיל לעבוד
- לחצן ▶ Power Dialer - בוחר רשימת "לידים חדשים שלי"

**10:00** - ליד מורכב
- בודק נכנס שיש לו רכב + נכס + הוצאה לפועל קודמת
- Eligibility card באדום: "לא ניתן להתקדם - 1 חוסם"

**11:30** - לקוח שאושר
- לחיצה על "Start Bidding" - 12 גופים נבדקים אוטומטית

- Best offer: כאל 50K @ 60 תשלומים, חיסכון ללקוח 320/חודש
- נציג מתקשר ללקוח, מציג את ההצעה (תסריט מוכן)
- Customer interest: "כן הלקוח מתקדם"
- Generate contract → WhatsApp ללקוח → חתימה תוך 10 דקות
- חשבונית יוצאת אוטומטית

**13:00** - הפסקת צהרים
- נציג נכנס ל"Tasks" - 2 משימות חדשות נדחפו על ידי AI
- "התקשר ליוסי כהן עוד 30 דק - אמר שיהיה זמין"
- "שלח BDI לנעמה (יום שלישי = הסכמה ניתנה)"

**15:00** - מנהל נכנס
- מקבל דשבורד: היום שלי - 8 שיחות, 3 לידים סגרתי, 2 contracts
- Manager dashboard: 12 נציגים פעילים, 47 שיחות, 11 conversions
- מנהל מאזין שיחה אחת (random sample) → AI score 87/100, summary טוב

**17:00** - סוף יום
- "Daily wrap-up" אוטומטי:
  - "סגרת 3 לידים, מעולה! 🎉"

  - "1 ליד מחכה ממך 4 ימים - שווה בדיקה"
- כפתור "End my day" - תזכורות מתחילות מחר

## רכיבי UX ייחודיים

### 1. Smart Search (⌘K)
- חיפוש לידים לפי שם / טלפון / ת.ז
- חיפוש פעולות ("חייג ליוסי", "שלח waטסאפ לדוד")
- חיפוש סטטוסים, תהליכים
- חיפוש משתמשים (להפניה)
- היסטוריה - מה חיפשתי לאחרונה

### 2. AI Co-pilot (sticky bottom-left)
תמיד נגיש - icon קטן עם נורה. לחיצה פותחת panel:
- "מה לעשות עכשיו?" - הצעה לפעולה
- "מה לשאול את הלקוח?" - שאלה בהתבסס על השלב
- "התנגדויות צפויות + תשובות"
- "סיכום השיחה האחרונה"
- "נתונים שחסרים בכרטיס"

### 3. Activity Timeline (sticky right)
- כל אירוע - איקון + טקסט + זמן
- סינון: שיחות / WhatsApp / SMS / status / notes
- Click → expand: שמע הקלטה, ראה תעתוק, ראה הודעה מלאה
- "Pin" אירוע חשוב לראש

### 4. Inline Quick Actions
בכל שדה - micro-actions:
- ת.ז → "בדוק תקינות" + "מלא פרטים מ-WOKi"
- טלפון → "חייג" / "WhatsApp" / "SMS"
- שם → "חפש כפילויות"
- כתובת → "Google Maps"

### 5. Smart Buttons
כפתורים שמשנים את עצמם:
- בהתחלה: "התחל לעבוד" (gradient ירוק)
- אחרי screening: "שלח לחתימה"
- אחרי חתימה: "התחל מכרז"
- אחרי מכרז: "הצג ללקוח"
- כל פעם הכפתור הבא נוצר אוטומטית בהתאם למצב

### 6. Notifications System
- Realtime via websocket
- Toast קטנים בפינה - לא בולמים
- Severity: info / success / warning / urgent
- Click → ניווט מיידי לליד הרלוונטי
- Settings: אילו התראות לקבל

### 7. Inbox-Zero Style
- לידים חדשים בולטים (badge + bg)
- אחרי טיפול - "Archive" אוטומטי לרשימה צידית
- "Empty inbox" - חוויית הצלחה ויזואלית

### 8. Speed Indicators
- מד "מהירות תגובה" שלך - בזמן אמת
- "ממוצע ענפי: 5 דקות. אתה: 3 דקות 🎉"
- gamification עדינה

### 9. Customer Sees You Mode
- כפתור "תצוגת לקוח" - מציג את הכרטיס כמו שהלקוח רואה
- מסתיר שדות פנימיים, מציג סיכום
- שימוש בפגישות פנים אל פנים

### 10. Dark Mode
- הרבה נציגים עובדים שעות ארוכות
- Dark mode מובנה (toggle)
- צבעי הירוק של בינגו עובדים יפה גם בכהה

## State Indicators - איך נציג יודע מצב

### Lead Card Header
```
[Avatar] שם הלקוח [PIPELINE: מחלקת החתמות] [STATUS: 🟢 ליד חדש]
         ת.ז 12345 · קל"ט 10/05/2026 · 4 ימים בתהליך זה
[🔥 Hot] [📞 Call now] [💬 WhatsApp] [⋯ More]
```

### Indicators:
- 🔥 Hot - ליד חמום (AI score > 70)
- ❄️ Cold - לא ענה ל-3 ניסיונות
- ⏰ Overdue - חולפים יותר מ-SLA
- ✅ Approved - אישור הלוואה
- 💰 Paid - שילם
- 🚨 Risk - תקלת compliance

## Error Handling

### עקרונות:
1. **לעולם לא לאבד עבודה** - autosave מתמיד
2. **שגיאות בשפה אנושית** - "השדה תעודת זהות לא תקין" (לא "Validation error E_INVALID_ID")
3. **תיקון מוצע** - "שכחת לציין סוג רכב? לחץ כאן להוסיף"
4. **Fallback graceful** - אם API נופל, working offline mode

- Time to interactive < 1.5s
- API response p95 < 300ms
- Search response < 200ms

## איך בודקים שהמערכת באמת UX טוב?

1. **Time-to-first-call** - נציג חדש מאיזה זמן יכול לעשות שיחה ראשונה?
2. **Task completion rate** - איזה % משלימים שלב שלם בלי שאלה?
3. **Errors per session** - שגיאות יוזר ממוצע
4. **NPS Internal** - נציגים מדרגים את המערכת
5. **Time to close lead** - השוואה לפני/אחרי
